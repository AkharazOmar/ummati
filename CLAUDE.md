# CLAUDE.md

Instructions for Claude Code when working on this project.

## Project overview

Ummati is a Flutter Islamic app (prayer times, Qibla compass, Quran reader, notifications).
Targets Android and iOS. Supports French, English, and Arabic (RTL).

## Key commands

```bash
# Run
flutter pub get
flutter run -d <device-id>

# Analyze
flutter analyze --no-fatal-infos

# Test
flutter test

# Generate localization files after editing .arb files
flutter gen-l10n

# Sync adhan sounds (after adding MP3 to assets/sounds/adhan/)
./scripts/sync_adhan.sh

# Build release APK
flutter build apk --release

# Build iOS
flutter build ios --release --no-codesign
```

## Architecture

- **State management**: Riverpod (StateNotifierProvider, FutureProvider, AsyncNotifierProvider)
- **Navigation**: GoRouter with StatefulShellRoute for bottom nav
- **Storage**: Hive (settings box, prayer_cache box)
- **Notifications**: flutter_local_notifications with zonedSchedule, custom sounds per prayer
- **Localization**: Flutter gen-l10n from .arb files (lib/l10n/). Always regenerate after editing .arb files.

## Important patterns

- Settings are persisted in Hive `settings` box via StateNotifierProviders in `settings_provider.dart`
- Prayer times are cached in Hive `prayer_cache` box, keyed by date + location mode
- Adhan sounds are loaded dynamically from `assets/data/adhan_list.json` (generated by `scripts/sync_adhan.sh`)
- Moroccan prayer method uses Aladhan `method=99` with `methodSettings=19,null,17`
- Location uses `getLastKnownPosition()` first for instant results, then falls back to GPS fix
- The `android/` directory is in .gitignore except files explicitly force-added
- Home screen widget uses `home_widget` package (Flutter) + native Kotlin `PrayerTimesWidget` (AppWidgetProvider)
- Widget data is pushed via `HomeWidget.saveWidgetData()` after prayer times are fetched, then `HomeWidget.updateWidget()` triggers native refresh
- Widget layout is in `android/.../res/layout/prayer_widget.xml` (RemoteViews XML, not Flutter)

## Testing

- Widget test uses a `_FakeBox` to mock Hive — no real storage needed
- CI runs on GitHub Actions: analyze + test on ubuntu, Android build on ubuntu, iOS build on macos

## Conventions

- Commit messages: `feat:`, `fix:`, `ci:`, `docs:` prefixes
- ARB files: always edit all 3 (app_en.arb, app_fr.arb, app_ar.arb) then run `flutter gen-l10n`
- No emojis in code unless user requests them
